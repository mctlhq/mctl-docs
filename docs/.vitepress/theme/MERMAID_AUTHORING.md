# Mermaid Authoring Notes

- Always use unique internal node IDs. Keep display labels separate from IDs.
- Avoid HTML in labels. Use Mermaid-safe plain text and line breaks only where supported.
- Prefer semantic classes and shared theme styling over inline one-off color tweaks.
- Group diagrams by domain boundaries first, then connect them with the main happy path.
- Keep labels short enough to remain readable both inline and in the expanded overlay.
